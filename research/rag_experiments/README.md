# Эксперименты с RAG для правил настольных игр

В данном разделе представлена базовая RAG-система (Retrieval Augmented Generation) для поиска и генерации ответов на вопросы о правилах настольных игр. Система использует Qdrant для векторного поиска и LlamaIndex для обработки документов и интеграции с LLM.

## Структура

- `src/`
  - `config.py` — конфигурация системы (параметры чанкинга, модели, LLM).
  - `chunking.py` — загрузка документов и разбиение на чанки (SentenceSplitter).
  - `indexer.py` — создание эмбеддингов и индексация в Qdrant.
  - `retriever.py` — семантический поиск релевантных чанков.
  - `generator.py` — генерация ответов с помощью LLM на основе найденного контекста.
- `scripts/`
  - `index_documents.py` — скрипт для индексации документов в Qdrant.
  - `build_manifest.py` — построение манифеста документов для индексации.
- `notebooks/`
  - `01_baseline.ipynb` — демонстрация и тестирование RAG-пайплайна (ретривер, генератор, анализ результатов).
- `docker-compose.yml` — конфигурация Qdrant контейнера.

> Данные для индексации берутся из `data/rules_texts_cleaned_good/` (результат работы `data_work/rules_cleaner.py`). Манифест документов находится в `manifests/index_manifest.csv`.

## Пайплайн индексации (`index_documents.py`)

Скрипт читает очищенные тексты правил из `data/rules_texts_cleaned_good/` и манифест из `manifests/index_manifest.csv`. Пока индексируются только документы на русском языке (`lang="ru"`). Затем:

1. Загружает документы и разбивает на чанки (`SentenceSplitter` с размером 512 токенов и перекрытием 80 токенов)
2. Создает эмбеддинги с помощью модели `intfloat/multilingual-e5-base`
3. Сохраняет векторы в Qdrant с метаданными:
   - `doc_id` — идентификатор документа
   - `game_titles` — список названий игр, связанных с документом
   - `lang` — язык документа
   - `source_file` — исходный файл

Чанкинг включает:
- Разбиение по предложениям с учетом размера чанка и перекрытия
- Сохранение метаданных для каждого чанка

## Пайплайн поиска и генерации

### Ретривер (`retriever.py`)

Семантический поиск релевантных чанков:
- Использует косинусное расстояние для поиска похожих векторов
- Возвращает топ-K результатов с оценками релевантности (scores)
- Поддерживает форматирование контекста для LLM

### Генератор (`generator.py`)

Генерация ответов на основе найденного контекста:
- Получает релевантные чанки от ретривера
- Формирует промпт с контекстом и вопросом
- Использует LLM для генерации ответа
- Поддерживает streaming генерацию

Системный промпт настроен на:
- Использование только информации из контекста
- Запрет на выдумывание деталей
- Краткие и точные ответы на русском языке

## Конфигурация

Все параметры в `src/config.py`:

| Параметр          | Значение по умолчанию           | Описание                                 |
|-------------------|---------------------------------|------------------------------------------|
| `EMBEDDING_MODEL` | `intfloat/multilingual-e5-base` | Модель эмбеддингов                       |
| `CHUNK_SIZE`      | `512`                           | Размер чанка в токенах                   |
| `CHUNK_OVERLAP`   | `80`                            | Перекрытие чанков в токенах              |
| `TOP_K`           | `5`                             | Количество результатов для ретривера     |
| `LLM_PROVIDER`    | `"ollama"`                      | Провайдер LLM: `"ollama"` или `"openai"` |
| `LLM_MODEL`       | `"qwen2.5:1.5b"`                | Модель LLM                               |
| `LLM_TEMPERATURE` | `0.0`                           | Температура генерации                    |
| `LLM_MAX_TOKENS`  | `512`                           | Максимум токенов для OpenAI              |

## Метаданные

- `doc_id` - идентификатор документа из манифеста, используется для связи чанков с исходными документами
- `game_titles` - список названий игр, связанных с документом (агрегируется из манифеста)
- `lang` - язык документа (ru/en)
- `source_file` — путь к исходному файлу в `data/rules_texts_cleaned_good/`

Каждый чанк в Qdrant содержит:
- Вектор эмбеддинга (768 измерений для `multilingual-e5-base`)
- Метаданные: `doc_id`, `game_titles`, `lang`, `source_file`
- Оценку релевантности (score) при поиске (косинусное расстояние)
